# Custom_CNN

## Project summary

- **Model:** Custom CNN segmentation baseline
- **Best validation IoU:** 0.3659 (epoch 10)
- **Final validation IoU:** 0.3597
- **Checkpoint size (approx.):** 89.9 MB
- **Parameters (approx.):** 7.86M

## Environment and dependencies

- **Python:** 3.10+ recommended (match your PyTorch build).
- **Hardware:** CUDA strongly recommended for training; CPU runs are possible but slow.
- **Install:**

```bash
cd Custom_CNN
pip install -r requirements.txt
```

- **PyTorch:** Install a `torch` / `torchvision` wheel that matches your OS and CUDA version from [https://pytorch.org/get-started/locally/](https://pytorch.org/get-started/locally/) if the defaults in `requirements.txt` are not suitable.

## Dataset layout and paths

Training scripts expect the **off-road** bundle under your **`duality project`** root (the directory that contains `Offroad_Segmentation_Training_Dataset`):

- **Train:** `Offroad_Segmentation_Training_Dataset/Offroad_Segmentation_Training_Dataset/train/` with `Color_Images/` and `Segmentation/` (paired files, same filenames).
- **Validation:** same structure under `.../val/`.

**Path resolution:** `train_segmentation5.py` searches upward from this folder for that layout. If auto-discovery fails, set either:

- `OFFROAD_DUALITY_PROJECT` — path to the `duality project` folder, or  
- `OFFROAD_SEGMENTATION_DATA_ROOT` — same intent (see script docstring).

**Testing:** `test_segmentation5.py` defaults to `Offroad_Segmentation_testImages/Offroad_Segmentation_testImages/`. Override with `OFFROAD_DUALITY_PROJECT` or `OFFROAD_SEGMENTATION_TEST_IMAGES` if needed.

## Step-by-step: train, test, and visualize

Run all commands from the `Custom_CNN` directory.

1. **Create environment and install dependencies** (see above).

2. **Train**

```bash
python train_segmentation5.py
```

3. **Evaluate / run inference on the test bundle**

```bash
python test_segmentation5.py
```

Optional: pass `--model_path` if your weights are not at the default `custom_cnn_best.pth` in this folder.

4. **Qualitative visualization (optional)**

`visualize5.py` colorizes **label mask** images from a folder. It currently sets `input_folder` to a **machine-specific absolute path** at the top of the file — edit that variable to your local `.../Offroad_Segmentation_testImages/Offroad_Segmentation_testImages/Segmentation` (or any folder of mask PNGs), then run:

```bash
python visualize5.py
```

## How to reproduce the reported results

The numbers in **Project summary** come from the **best / final validation metrics** written during training.

1. Run `python train_segmentation5.py` with the same dataset and a similar stack (PyTorch + CUDA).
2. After training, open **`train_stats5/evaluation_metrics.txt`**. Compare **Best Val IoU** and **Final Val IoU** to the summary above.
3. For a held-out **test** run, use `python test_segmentation5.py` and read **`test_evaluation_metrics.txt`** (or the path printed in the console) under the script’s output directory.

**Note:** Exact IoU values can differ slightly across runs due to random initialization, non-deterministic CUDA ops, and library versions. The README figures are the reference from the original submission run.

## Expected outputs and how to interpret them

| Artifact | Meaning |
|----------|--------|
| `custom_cnn_best.pth` | Best checkpoint (by validation IoU) saved in this folder. |
| `train_stats5/evaluation_metrics.txt` | Training summary: final/best val IoU, Dice, accuracy, per-class IoU, epoch table. |
| `train_stats5/` charts | Optional plots saved alongside metrics (if generated by the script). |
| `test_segmentation5.py` outputs | Colored predictions, comparison figures, and **`test_evaluation_metrics.txt`** with mean / per-class IoU on the test folder (when masks are present). |

**IoU (Intersection over Union):** mean IoU is the average of per-class IoUs; higher is better (max 1.0). Per-class bars highlight which terrain classes are easy vs. hard for the model.
